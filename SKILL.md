# 🏛️ 官僚帝国 - Bureaucracy Empire Skill

## 📜 概述

模拟古代三层六部制的组织化任务管理体系，通过角色扮演实现复杂任务的分解、讨论和执行。

---

## ⚡ 安装与部署（必读！）

### ⚠️ 关键提示

**这个技能能否工作，完全取决于 AGENTS.md 中是否有触发词配置！**

没有配置 → 技能永远不会被激活  
配置正确 → 每次发送 `🍀圣旨到！` 都会启动完整流程

---

### 📋 步骤 1: 克隆仓库

```bash
git clone https://github.com/miaouai/bureaucracy-empire.git
cd bureaucracy-empire
```

### 📋 步骤 2: 复制到 active_skills

```bash
cp -r . /app/working/active_skills/bureaucracy-empire/
```

### 📋 步骤 3: 配置 AGENTS.md（关键！）

#### 方案 A: 自动配置（推荐）

```bash
# 运行一键配置脚本
bash /app/working/install_bureaucracy_empire.sh
```

#### 方案 B: 手动添加（如果自动脚本不适用）

编辑 `/app/working/AGENTS.md`，找到或添加以下章节：

```markdown
### ⚡ 技能触发规则

**强制触发关键词**（检测到必须立即加载对应技能）：

| 关键词 | 触发技能 | 说明 |
|--------|----------|------|
| `🍀圣旨到！` | `bureaucracy-empire` | 启动官僚帝国任务管理系统，10 人内阁议政 + 六部执行 |

**触发协议**：
1. 扫描用户消息是否包含触发关键词
2. 匹配成功 → **立即加载**对应技能
3. 严格按照技能文档执行流程
4. 禁止跳过或忽略触发词
```

**具体位置**: 
- 如果 AGENTS.md 已有其他技能触发规则，追加到新行
- 如果没有，在文件末尾新建 `### ⚡ 技能触发规则` 章节

### 📋 步骤 4: 验证安装

```bash
# 检查文件完整性
ls /app/working/active_skills/bureaucracy-empire/SKILL.md

# 检查 AGENTS.md 配置
grep "🍀圣旨到" /app/working/AGENTS.md
```

预期输出应显示文件存在，且 grep 返回包含 trigger 词的行。

### 🔧 故障排查

| 问题 | 解决方案 |
|------|---------|
| 输入"🍀圣旨到！"没反应 | 检查 AGENTS.md 是否有触发词配置 |

## 🔄 工作流程

### Stage 1: 圣旨接收
- 检测 `🍀圣旨到！` 触发词
- 读取原始任务
- 记录到 `logs/圣旨.log`

### Stage 2: 内阁议政 (核心讨论)
- 依次激活 10 位顾问角色
- 从不同角度分析问题
- 争论、补充、完善思路
- 产出《内阁议政纪要》

### Stage 3: 尚书定策
- 解析议政结果
- 识别关键子任务
- 匹配执行部门
- 生成《尚书发配令》

### Stage 4: 六部执行
- 各部首脑领取任务
- 调用专业技能执行
- 记录执行过程
- 产出《XX 部奏折》

### Stage 5: 御前汇报
- 汇总各部成果
- 形成完整方案
- 向主公汇报
- 等待旨意（修改/通过）

## 🧬 自进化机制

每次执行后自动记录：
- ✅ 哪些角色最有价值？
- ❌ 哪个环节卡住了？
- 💡 有什么改进点子？

定期（每日 9 点 cron）自动更新角色设定和流程模板。

## 🎭 使用示例

**主公说：**
> 🍀圣旨到！朕想在一个月内找到新工作，你给我出个全套方案

**系统响应：**

1. **内阁开议** → 10 位顾问各抒己见
2. **尚书发令** → 拆分为：简历优化、面试准备、职位搜索、人脉拓展等任务
3. **六部行动** → 
   - 吏部：梳理工作经历、能力评估
   - 礼部：撰写简历、模拟面试话术
   - 工部：搭建个人作品集网站
   - 兵部：调研竞品公司、薪资谈判策略
   - ...
4. **回禀圣上** → 提交 30 天冲刺计划表

## ⚠️ Token 节省策略

1. **简化版模式**: 只启用关键 3-5 个角色
2. **缓存记忆**: 重用之前的类似任务讨论结果
3. **渐进披露**: 主公追问时才展开细节
4. **批量处理**: 多个小任务合并一次流转
